## Layer Breakdown

### L1: Category Gate
**Purpose:** Kill noise at the door. Only allow categories with fundamental analytical basis.

**Filters:**
- Exclude: Pop Culture, Meme, Entertainment
- Allow: Politics, Crypto, Business, Science, Global Affairs, Sports, Economics
- Markets with no category pass through (some API markets lack category metadata)

**Code:** `l1_category.go`
**Config:** `category_gate.excluded_categories`

---

### L2: Liveness Check
**Purpose:** Is this market alive and breathing?

**Filters:**
- `active == true`
- `closed == false`
- `endDate > now`
- `liquidity > $1,000`
- `volume_24h > $500`

**Code:** `l2_liveness.go`
**Config:** `liveness.min_liquidity`, `liveness.min_volume_24h`

**Key change:** These filters are now in the scanner pipeline (`l2_liveness.go`), NOT in the Gamma API fetch call (`gamma.go`). This makes the API layer dumb and the filter logic modular/testable.

---

### L3: Quality Gate
**Purpose:** Is this market worth analyzing?

**Filters:**
- Price parsing (YES/NO from JSON string)
- Horizon: 3-30 days to resolution
- Golden Zone: YES or NO in $0.20-$0.40 (uses last_trade_price for now; L4 will use VWAP)
- Liquidity tier: >$5K minimum

**Code:** `l3_quality.go`
**Config:** `quality.horizon_min_days`, `quality.horizon_max_days`, `golden_zone.min/max`

**Note:** L3 still uses `last_trade_price` because we don't have order book data yet. L4 (in M1.3+M1.2b) will recalculate with VWAP.

## Output Files

Each scan produces:

1. **JSON:** `scan-TIMESTAMP.json` — L3 survivors with market data

   - Filter configuration
   - Pipeline results (L1/L2/L3 pass/reject counts)
   - Rejection breakdown by layer and reason
   - L3 survivors by tier with clickable links
3. **Rejects:** `scan-TIMESTAMP-rejects.json` — Full rejection log with layer, reason, market data
